4  Task One

This chapter describes a summative exercise for this class.

4.1 Assessment information

You will be given a dataset and a finished report on that data. Your task is to write the code that produces the report.

The solution will be released after the deadline.

This is part of a weekly set of exercises and counts for a small part of your final grade

You should submit your work by pushing your finished repository to Github Classrooms

Please read all the assessment information before attempting the exercise.

The practice report will not be individually graded. Instead, we will release the solution file so that you can compare your code to our solution as generic feedback. If you would like to discuss any aspect of the report, you can attend office hours with me.

4.2 Intended Learning Outcomes

  • Demonstrate the ability to use R for data wrangling and visualisation

  • Demonstrate the ability to use Quarto for reproducible reports

  • Demonstrate the ability to use Github for collaborative coding

4.3 Evaluation

  • Demonstrate the ability to evaluate an output to understand the underlying data wrangling and analysis

4.4 Details and files

Follow the provided link to Github Classrooms - start a team or join an existing one (maximum group size of three).

You will need a Github account to complete this assignment

Once you have collected your assignment it will generate a private repository for you and your teammates. It contains four things:

  • The first is the dataset: covid_data.csv

  • The second is the finished report that you need to backwards engineer from the data: report_output.html.

  • The third is a .Rproj file

  • The fourth is a README - this is a text (.md) file that is the front page of your repository

There are multiple ways to achieve the same outcome as the finished report when coding. You do not have to use the same solutions that we did; what matters is that the output is the same.

All plots, in the report should be full reproducible.

For any visuals, you should get as close to the example report as possible, although we recognise that it may be difficult to know the exact theme or colours used without seeing the code.

In order to replicate the report you will need the following skills:

  • Data visualisation

  • Data wrangling

  • Github commits

  • Github branch management

Almost everything you need to do generate this report will have been covered at least once in the workbook, but you should also expect to need to Google some things – being able to search for alternative solutions is such a key skill for programming.

  • Each member of the group should set up a separate branch in the github repository, and each person should work on producing a single figure.

  • Add comments (using #) to explain your code.

  • When your contribution is finished submit a pull request to the main branch, ask another member of the group to review your code.

  • Knit the final .qmd file to produce your copy of the report.

  • Work together and help each other.

4.5 Support

This assessment will require you to use the skills you have learned from the previous module BIO-5027A and our introduction to Github.

Google and Stack Overflow will help you a lot. Remember to include the names of packages and functions to help ensure the results are closer to what you’re trying to do.

Do not use AI to write your code. We’re going to do that in the next Chapter.

Do not use Renv for this assignment - although it helps snapshot R versions and packages, the lockfile can cause merge conflicts when working collaboratively, and so we will make things simpler for now.

4.6 Helpful hints

A few hints to help you on your way:

  • This dataset is bigger than the ones you’ve used before, which means you might find it takes R a little longer to complete certain tasks.

  • You will need to use Spinu et al. (2024) to handle dates properly

  • You will need Wickham (2023) to handle factors and ordering

  • You will need group_by() and summarise() for group-specific calculations

  • You will need if_else() or case_when() to conditionally count cases

  • You will need pivot_longer() or pivot_wider() to reshape data

4.7 Why am I being assessed like this?

This report serves as a practice for the full report, which will be due at the end of the semester. The practice report gives you the intended output so that you can focus on your coding skills, rather than deciding how to summarise and present the data. You will need to make these decisions for the full report.

Whilst most of the skills required for the practice report have been explicitly taught, there are minor bits of the coding that will require you to look up solutions independently. This will be good practice for the final summative assessment and for applying your coding skills to your own work.

Spinu, V., Grolemund, G., & Wickham, H. (2024). Lubridate: Make dealing with dates a little easier. https://lubridate.tidyverse.org
Wickham, H. (2023). Forcats: Tools for working with categorical variables (factors). https://forcats.tidyverse.org/
